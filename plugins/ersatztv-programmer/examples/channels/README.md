# Per-bucket channel.json templates

Five `channel.json` starters, one per bucket in the 75/5 architecture.
Setup wizard copies the matching template to
`${CHANNELS_DIR}/{N}/channel.json` and substitutes `{N}` with the channel
number when provisioning a new channel.

| File | Bucket | Profile |
| :--- | :--- | :--- |
| `core.channel.json` | core | 1080p HEVC 10-bit, 2 Mbps, full loudness normalization |
| `rotating.channel.json` | rotating | Same A/V profile as core (only the schedule differs) |
| `music.channel.json` | music | 720p HEVC 8-bit + 256 kbps audio, looser loudness |
| `live.channel.json` | live | 720p H.264 pass-through, no loudness normalization |
| `experimental.channel.json` | experimental | Matches core so format experiments can borrow any content |

Per-template notes live in the `_comment` field of each JSON. The schema
forbids extra fields anywhere else, so the comment is the only inline
documentation surface.

The plugin's setup wizard also writes:

- `channels/{N}/playout/` — empty directory the daily refresh writes into.
- `channels/{N}/logo.png` — generated by `tools/render-logo.py`.

Logos are referenced from XMLTV (`<icon>`), not from `channel.json` — the
ErsatzTV Next channel schema doesn't have a logo field today. See
`skills/setup/channel-logos.md`.
